# AI-Powered-Cyber-Threat-Intelligence-Platform-using-MISP-and-Multi-Source-IOC-Automation
Developed a comprehensive Cyber Threat Intelligence (CTI) platform using MISP to centralize, manage, and operationalize threat intelligence in a production-like environment. The project involved deploying MISP using Docker containers, ensuring scalability, service isolation, and secure communication through proper configuration of backend services such as MySQL, Redis, and Nginx. This setup provided a reliable infrastructure for ingesting, storing, and analyzing large volumes of threat data.

A key component of the project was building a high-confidence intelligence event focused on APT28, a well-known state-sponsored threat actor. The event was enriched with structured tags, MITRE ATT&CK mappings, and malware references to provide both tactical and strategic context. Indicators of Compromise (IOCs), including IP addresses, domains, URLs, and file hashes, were extracted from trusted OSINT sources and transformed into structured attributes within MISP, enabling efficient correlation and detection.

To enhance data coverage and ensure continuous intelligence updates, multiple external threat feeds were integrated, including CIRCL OSINT and Abuse.ch services such as URLhaus, ThreatFox, and MalwareBazaar. These feeds were configured with caching and automated synchronization, transforming the platform into a dynamic system capable of ingesting real-time threat data. Background job monitoring ensured successful ingestion and processing of feed data without impacting system performance.

In addition, a custom Python-based ETL pipeline (unify_feeds.py) was developed to aggregate and standardize IOCs from diverse sources, including MISP exports, CSV feeds, and JSON-based threat intelligence platforms. The pipeline performed normalization of indicator types, removal of duplicates, and validation of data integrity, producing clean and consistent datasets in CSV and JSON formats. This unified dataset was then re-imported into MISP using FreeText Import, bridging external processing with internal intelligence management.

Finally, visualization was implemented using Grafana, where SQL queries were used to analyze IOC distribution and provide insights into the most common threat indicator types. This end-to-end solution demonstrates the ability to collect, process, enrich, and visualize cyber threat intelligence, making it actionable for Security Operations Center (SOC) environments and real-world defensive use cases.
